#### 开发人员 (ROLE_DEVELOPER)

## 技术实现细节

### 1. 雪花ID使用
所有表的主键都使用雪花ID（String类型）：
- 用户ID：`1`, `2`, `3`, ...
- 角色ID：`1`, `2`, `3`, ...
- 部门ID：`1`, `2`, `3`, ...

### 2. 审计字段
所有表都包含审计字段：
- `create_time` - 创建时间
- `update_time` - 更新时间
- `create_user` - 创建人
- `update_user` - 更新人
- `deleted` - 逻辑删除标记

### 3. 密码加密
所有用户密码使用BCrypt加密：
```
原始密码：123456
加密后：$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iKTV2UiC
```

## 后续维护

### 1. 新增用户
```sql
-- 新增用户示例
INSERT INTO sys_user (id, username, password, nickname, email, phone, dept_id, status) 
VALUES ('新ID', '新用户名', '加密密码', '昵称', '邮箱', '电话', '部门ID', 1);
```

### 2. 修改权限
```sql
-- 修改角色权限示例
DELETE FROM sys_role_menu WHERE role_id = '角色ID';
INSERT INTO sys_role_menu (role_id, menu_id) 
SELECT '角色ID', id FROM sys_menu WHERE perm_key IN ('权限列表');
```

### 3. 新增角色
```sql
-- 新增角色示例
INSERT INTO sys_role (id, code, name, description, status) 
VALUES ('新ID', 'ROLE_NEW', '新角色', '角色描述', 1);
```

通过这套完整的数据，系统具备了完整的用户权限管理体系，支持多角色、多部门的复杂权限控制。
